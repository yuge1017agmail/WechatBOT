# 知识库

把产品资料、卖点、报价规则、交付说明、售后政策等 Markdown 文档放在这个目录下。系统会在生成回复前检索这里的 `.md` 文件，只把命中的少量片段交给大模型。

推荐结构：

```text
knowledge/
  products/
    花瓣沙发.md
    云朵床.md
  policies/
    交付说明.md
    售后说明.md
```

文档建议这样写：

```md
# 花瓣沙发

## 核心卖点

- 造型柔和，适合法式、奶油风、现代轻奢空间。
- 坐感偏软，但仍有支撑，不是塌陷型。
- 可选布艺、科技布、真皮。

## 适合客户

- 喜欢柔和曲线感。
- 希望客厅更有设计感。
- 不想要太硬朗的传统沙发。

## 回复注意

客户问价格时，先问尺寸和材质，不要直接报死价。
```

写法建议：

- 一个产品或一个政策写一个文件。
- 标题尽量包含客户会说的名字，比如“花瓣沙发”“云朵床”。
- 专业资料写事实，不要写泛泛的销售话术。
- 对不确定的价格、库存、交期，写清楚“需要确认”。
