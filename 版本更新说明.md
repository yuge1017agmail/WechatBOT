## 2026-04-27

本次迭代围绕“更稳定地自动回复微信私聊”和“让回复更像真人销售表达”展开，主要更新如下。

### 1. 模型配置支持

- `.launcher-config.json` 新增 `MODEL_NAME` 配置项。
- 启动器会读取 `MODEL_NAME` 并传给运行进程。
- 大模型请求会使用配置里的模型名，不再只能使用代码中写死的模型。
- 修复 DeepSeek 接口路径兼容问题：
  - DeepSeek baseURL 使用 `https://api.deepseek.com`
  - 请求路径自动使用 `/chat/completions`
  - 模型示例为 `deepseek-v4-flash`

### 2. 启动弹窗优化

- 启动时拆成两步选择：
  1. 选择发送模式：`仅粘贴不发送` 或 `自动发送`
  2. 选择消息节奏：标准、快速、稳妥
- 消息节奏预设：
  - 标准：8 秒轮询，10 秒合并等待，30 条历史
  - 快速：5 秒轮询，8 秒合并等待，30 条历史
  - 稳妥：15 秒轮询，15 秒合并等待，50 条历史
- 启动日志会显示当前发送模式和消息节奏。

### 3. 多消息处理优化

- 将原先固定 30 秒轮询改为可配置轮询。
- 新增同一用户消息合并等待机制：
  - 同一用户短时间连续发送多条消息时，系统会等待一段时间。
  - 等该用户不再继续发消息后，再合并理解并生成一次回复。
- 后台理解和生成回复改为串行处理，避免多个用户同时触发时抢占处理链路。
- 保留 `output/*.md` 任务文件机制，用于发送缓冲、失败归档和人工排查；这一步不增加模型 token 成本。

### 4. 微信 CLI 调用稳定性

- `wechat-cli` 调用从 shell 字符串拼接改为参数数组方式。
- 联系人名中包含空格、竖线等特殊字符时，读取历史记录更稳定。
- 增强聊天历史解析逻辑，兼容 `wechat-cli history --format json` 返回对象或数组的情况。

### 5. 人设与回复风格

- 新增 `人设.md`。
- AI 每次生成回复前都会读取 `人设.md`。
- 现在可以通过修改 `人设.md` 调整：
  - 身份设定
  - 语气
  - 用词习惯
  - 回复边界
  - 禁止事项
- 目标是减少 AI 味和客服腔，让回复更接近真实微信表达。

### 6. 知识库检索

- 新增 `knowledge/` 知识库目录。
- 新增 `knowledge-search.ts` 本地 Markdown 检索模块。
- 客户咨询专业问题时，系统会先检索知识库，再结合：
  - 客户问题
  - 最近聊天记录
  - `人设.md`
  - 命中的知识库片段
  生成最终回复。
- 当前使用本地关键词检索：
  - 不调用额外模型
  - 不产生 embedding 成本
  - 每次最多取前 5 个相关片段进入 prompt
- 没有检索到相关资料时，系统会要求模型不要编造专业细节，而是自然说明“我确认下再回你”。

### 9. 微信粘贴可靠性修复

- 修复任务进入 `output/sent/`，但微信输入框没有实际粘贴/发送的问题。
- 原因是旧版 `deliver_wechat_message.applescript` 只激活微信后直接执行 `Cmd+V`，如果焦点不在输入框，脚本不会报错，分发器会误判成功。
- 新版发送脚本会：
  - 根据微信窗口位置点击聊天输入区
  - 粘贴回复内容
  - 复制输入框内容并校验是否包含本次回复
  - 校验通过后才按发送模式继续
  - 校验失败时抛错，让任务进入 `output/failed/`，避免误归档到 `sent/`
- 根据实测结果，将输入区点击坐标从底部上方 78px 调整为底部上方 35px，避免点到微信底部工具栏导致第二次粘贴失败。

### 10. 微信账号切换检测

- 启动器新增 `wechat-cli` 账号绑定检查。
- 启动前会读取 `~/.wechat-cli/config.json` 中的 `db_dir`，并扫描微信容器里的账号数据库目录。
- 如果发现 `wechat-cli` 当前绑定目录和微信最近活跃账号目录明显不同，会停止启动并弹窗提示。
- 提示会引导在当前微信账号登录状态下执行：
  - `wechat-cli init`
- 这样可以避免更换微信账号后，后台仍读取旧账号数据库，导致“识别不到新消息”。
- 新增测试覆盖：
  - 多账号数据库目录按活跃时间排序
  - 当前账号一致时正常启动
  - 检测到账号切换时拒绝启动
  - 缺少 `wechat-cli` 配置时提示初始化

### 11. 已打开会话的重复发送修复

- 修复同一发信人第一次回复成功、第二次回复时粘贴校验失败的问题。
- 原因是准备会话脚本过去无条件使用 `Cmd+F` 搜索联系人；当目标会话已经打开时，微信可能把焦点留在搜索/查找状态，导致后续粘贴没有进入聊天输入框。
- 新版准备脚本改为：
  - 先按 Esc 清理可能残留的搜索状态
  - 直接点击微信左上角全局搜索框
  - 搜索并进入目标会话
  - 再按 Esc 退出搜索状态
  - 最后把焦点放回聊天输入区
- 发送脚本在粘贴前也会先按 Esc，再重新聚焦输入区，降低重复发送时焦点错位的概率。
- 继续优化搜索框定位：
  - 搜索框点击位置从边缘调整到搜索框中心
  - 连续点击两次，提高焦点进入全局搜索框的成功率
  - 粘贴回复前会先清空输入框草稿，避免误粘的联系人名混入回复
  - 准备会话阶段不再按 Enter 选择搜索结果，改为鼠标点击第一条搜索结果，避免搜索框未聚焦时把联系人名直接发送出去

### 12. 服务号与公众号过滤

- 新增真实私聊判断逻辑，只处理需要回复的个人私聊。
- 自动忽略：
  - 群聊
  - `brandservicesessionholder` / `brandsessionholder`
  - `gh_` 开头的公众号账号
  - 常见通知/服务账号
  - 显示名为 `服务号` 或 `订阅号` 的入口
- 轮询日志会显示已忽略的群聊/服务号消息数量。

### 当前建议用法

1. 修改 `.launcher-config.json`，配置 API key、baseURL 和模型名。
2. 修改 `人设.md`，让回复贴近自己的微信表达。
3. 将产品资料、卖点、售后政策等写入 `knowledge/` 下的 Markdown 文件。
4. 双击 `Start WeChat AI.command`。
5. 首次建议选择：
   - 发送模式：`仅粘贴不发送`
   - 消息节奏：`标准`
6. 确认回复内容、窗口定位和知识库引用效果稳定后，再切换到 `自动发送`。
